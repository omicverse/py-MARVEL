# Plate Phase 1 R References

This directory stores the R MARVEL reference inputs and outputs for Phase 1, and Python tests should read from here.

The contract is:

- `README.md` documents the directory purpose.
- `manifest.json` is the committed canonical contract artifact generated by `scripts/export_plate_phase1_reference.R`.
- `manifest.json` must declare `phase = "plate_phase1"`.
- `manifest.json` must list the 10 required functions in order.
- `inputs/*.tsv` holds the shared 4-sample reference dataset used by both the R exporter and the Python comparison tests.
- `object_*.tsv`, `alignment_*.tsv`, `compute_psi_se.tsv`, `compute_psi_posterior_se.tsv`, `assign_modality_core.tsv`, `count_events_min1.tsv`, and `compare_values_gene.tsv` are exported from canonical R MARVEL on those shared inputs.
- `assign_modality_core.tsv` intentionally includes only the stable scaffold columns. On this small dataset, R and Python disagree on final modality labels, so the comparison suite avoids brittle exact-label assertions there.
- `manifest.json` includes `comparison_status`, so exact versus partial coverage is explicit. `PropModality` remains partial coverage in Task 7.
- `CompareValues` remains partial coverage in Task 7: exact R parity is exported only for the gene-level branch via `compare_values_gene.tsv`, while the splicing branch is still covered by Python-side contract tests rather than exact R parity.
